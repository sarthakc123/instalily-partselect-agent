## Reading order for a new engineer

1. [../README.md](../README.md) — get it running, try the demo queries.
2. [../architecture.md](../architecture.md) — understand the agent topology, the five tools, why the KG is structured and not GraphRAG.
3. [tools.md](tools.md) — see each tool's contract and example invocation.
4. [api.md](api.md) — see the SSE event union if you're touching the frontend or building a different client.
5. [security.md](security.md) — internalize the hard rules before changing user-facing copy or PII handling.
6. [eval_methodology.md](eval_methodology.md) — add a case before changing behavior.

## Reading order for a reviewer

1. [../README.md](../README.md) — context + how to run + what's built.
2. [slides.pdf](slides.pdf) — design choices + business impact + scaling roadmap (the pitch).
3. The Loom video (link in [../README.md](../README.md)) — live demo of the four query patterns.
4. [eval_results.md](eval_results.md) — every assistant reply verbatim, including the compound query.
5. [../architecture.md](../architecture.md) — go deep on the design.
